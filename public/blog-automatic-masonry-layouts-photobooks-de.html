Designsysteme / Field Note 001

Das Web beherrscht automatische Masonry-Layouts. Warum Fotobücher nicht?

Browser können ungleich große Bilder seit Jahren zu überzeugenden Rastern anordnen. Bei Fotobuchsoftware schieben wir Erinnerungen noch immer von Hand in Kästen. Der Grund ist einfach: Eine Seite braucht mehr als eine passende Anordnung. Sie braucht Urteilsvermögen.

Graphic Meat12. Juli 20268 Min. Lesezeit

Masonry-Layout ist eine jener Webideen, die im Rückblick selbstverständlich wirken. Man gibt Bilder mit unterschiedlichen Proportionen vor, wählt eine Spaltenbreite und lässt die Elemente den verfügbaren Raum ausfüllen. Pinterest machte das Muster bekannt. JavaScript-Bibliotheken machten es zum Standard. Modernes CSS bringt es nun näher an die Plattform selbst.

Warum fühlt sich das Erstellen eines Fotobuchs nach mehr als einem Jahrzehnt Masonry-Raster auf Bildschirmen noch immer wie Desktop-Publishing im Kleinformat an?

Im Web ist Leerraum billig

Eine Webseite kann immer weiter wachsen. Wird eine Spalte länger, wird das Dokument einfach höher. Die Lesenden scrollen. Ein Masonry-Algorithmus muss nur eine lokal sinnvolle Entscheidung treffen: das nächste Element dort platzieren, wo die kleinste Lücke bleibt.

Eine gedruckte Doppelseite ist ein geschlossenes System. Sie hat feste Breite und Höhe, einen Bundsteg, den man nicht unbedacht überqueren sollte, Beschnitt an den Rändern und eine begrenzte Zahl von Fotos, die alle hineinpassen müssen. Unterhalb der Falz gibt es keine „weitere Seite“.

Ein Browser optimiert Raum. Ein Fotobuch muss Aufmerksamkeit optimieren.

Fotos sind keine austauschbaren Rechtecke

Layout-Engines sehen Breite, Höhe und Seitenverhältnis. Menschen sehen ein Gesicht am Bildrand, einen Horizont, der gerade bleiben muss, ein stilles Bild, das Raum braucht, oder genau das Foto, um das sich die ganze Reise dreht.

Ein technisch perfekter Beschnitt kann das Motiv entfernen. Ein mathematisch ausgewogenes Raster kann einem Kassenbon dasselbe visuelle Gewicht geben wie einem Hochzeitsporträt. Zwei Bilder, die Sekunden nacheinander aufgenommen wurden, gehören vielleicht zusammen; zwei mit identischem Seitenverhältnis vielleicht nicht.

Die fehlende Ebene ist die Bewertung

Das nützliche Modell ist nicht eine einzige raffinierte Masonry-Regel, sondern ein Generator mit anschließendem Kritiker. Zuerst entstehen viele gültige Anordnungen: bekannte Vorlagen, rekursive Seitenpartitionen, Reihen mit gemeinsamer Höhe, Spalten mit gemeinsamer Breite. Danach wird jeder Kandidat anhand der Eigenschaften bewertet, die auch ein Designer bemerken würde.

  • Beschnittkosten: Wie viel von jedem Foto verschwindet?
  • Motivsicherheit: Bleiben Gesichter und wahrscheinliche Hauptmotive erhalten?
  • Balance: Wirkt das visuelle Gewicht über den Bundsteg hinweg beabsichtigt?
  • Rhythmus: Sind aufeinanderfolgende Doppelseiten abwechslungsreich, ohne chaotisch zu werden?
  • Hierarchie: Bekommt das Hauptbild genügend Raum?
  • Drucksicherheit: Werden Beschnitt-, Schnitt- und Bundstegvorgaben eingehalten?

Das höchstbewertete Layout ist nicht „richtig“. Es ist der beste Vorschlag, den die Software mit den vorhandenen Informationen machen kann.

Automatisch sollte einen starken ersten Entwurf bedeuten

Fotobuchprogramme wählen oft zwischen zwei schlechten Versprechen: Entweder ist alles manuell, oder „KI“ erstellt ohne Zutun ein fertiges Buch. Echte Gestaltung liegt dazwischen.

Automatisierung sollte das Problem der leeren Seite beseitigen. Sie sollte 180 Fotos in Sekunden in einen stimmigen ersten Entwurf verwandeln, die Originaldateien bewahren und jede Entscheidung reversibel halten. Die Nutzenden bleiben Redakteure: dieses Bild hervorheben, jene beiden zusammenhalten, diese Doppelseite neu erzeugen, jenen Beschnitt fixieren.

Warum jetzt?

Die einzelnen Bausteine sind ausgereift. Geräte können Gesichter lokal erkennen. Fotomediatheken enthalten bereits Zeitstempel, Orte und Favoriten. Layouts lassen sich schnell genug erzeugen, um interaktiv Hunderte Kandidaten zu prüfen. Native Apps können all das erledigen, ohne persönliche Fotos auf einen Server hochzuladen.

Dieser letzte Punkt ist wichtig. Familienarchive sind intim. Eine gute automatische Layout-Engine muss nicht zu einem weiteren Clouddienst werden. Sie kann neben der Mediathek auf dem Gerät des Besitzers laufen und vergisst nichts, weil sie nie etwas weggenommen hat.

Woran wir arbeiten

PhotoBooks kombiniert Vorlagen und generative Seitenpartitionierung und bewertet die Ergebnisse anschließend mit einem gemeinsamen Bewertungssystem. Ziel ist nicht, einen menschlichen Designer nachzuahmen. Wir möchten allen den Teil gestalterischer Erfahrung geben, den Software verlässlich liefern kann: Proportion, Konsistenz, Druckvorgaben und einen brauchbaren Ausgangspunkt.

Das Web hat bewiesen, dass Rechtecke sich selbst anordnen können. Fotobücher brauchen den nächsten Schritt: Rechtecke, die verstehen, dass sie Erinnerungen tragen.

Probieren Sie die Layout-Engine aus. PhotoBooks ist für macOS kostenlos, funktioniert offline und exportiert druckfertige PDFs.
