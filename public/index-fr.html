// Qui nous sommes

Graphic Meat est un studio de produits indépendant, en activité depuis 2013. Nous construisons de petits outils pointus qui respectent vos données et font bien une chose. Certains sont payants par abonnement, d'autres sont gratuits et open source : le modèle suit le produit, tandis que l'objectif reste le même : un logiciel utile et un studio capable de continuer à le créer. Bien fait n’est pas une exigence.

// Comment nous construisons

Les petits outils méritent toujours une véritable ingénierie. Nous nous appuyons sur les modèles déjà prouvés par le monde du logiciel et les associons à l'IA pour passer d'un référentiel vide à une expédition rapide du produit, sans le désordre qui accompagne généralement la rapidité.

  • Modularité

    Petites pièces autonomes avec coutures claires. Chaque partie fait un travail, donc les fonctionnalités apparaissent et les bogues restent encadrés.

  • Piloté par les tests

    Écrivez d'abord le test, puis le code qui le réussit. Le comportement est épinglé avant sa livraison et reste épinglé à mesure que le code se développe.

  • Conçu pour tester

    Code façonné pour pouvoir être vérifié : fonctions pures, dépendances injectées, pas d'état caché. Si c'est difficile à tester, c'est qu'il est mal conçu.

  • KISS

    La chose la plus simple qui fonctionne à chaque fois. Pas d’abstractions spéculatives, pas d’astuce à décoder à 3h du matin. Le code ennuyeux est une fonctionnalité.

  • SOLID

    Les principes de conception éprouvés maintiennent les responsabilités propres et les changements peu coûteux, de sorte que la base de code se plie au lieu de se briser.

  • Accéléré par l'IA

    L'IA fait le gros du travail en matière d'échafaudages, de refactorisations et de révisions - sous le jugement humain - de sorte que la qualité augmente tandis que le temps de construction diminue.
